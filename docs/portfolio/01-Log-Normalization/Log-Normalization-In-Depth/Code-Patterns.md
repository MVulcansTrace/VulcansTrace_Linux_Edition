## The Security Problem

Linux firewall logs are generated by the kernel as unstructured text. Each line contains key=value pairs in inconsistent positions, with no schema enforcement, no field ordering guarantees, and no validation. Downstream detectors require a clean, typed, validated event stream — they cannot afford to re-parse and re-validate every field on every event. The normalization layer must bridge this gap with patterns that are reliable, testable, and resistant to both accidental corruption and deliberate evasion.

---

## Implementation Overview

| Pattern | Where Used | Purpose |
|---|---|---|
| Multi-regex field extraction | Both parsers | Extract individual fields without brittle monolithic regexes |
| Compiled static regex fields | Both parsers | Amortize compilation cost across all lines |
| Fail-soft per-line try/catch | Both parsers | Skip bad lines without aborting the parse |
| Action derivation from context | Both parsers | Infer firewall action when no explicit field exists |
| Immutable validated event model | UnifiedEvent | Prevent invalid data from reaching detectors |
| Extensible metadata dictionary | UnifiedEvent | Carry format-specific fields without schema changes |
| Input size guard | LogNormalizer | Prevent resource exhaustion from oversized inputs |
| Error sanitization | Both parsers | Prevent log injection through error messages |
| Null logging default | Both parsers + LogNormalizer | Work without external logging infrastructure |

## How It Works (Technical)

### Multi-Regex Field Extraction

Each parser applies individual regexes to every line rather than one monolithic pattern. This avoids cascade failure — if the MAC regex fails, SRC, DST, PROTO, and ports are still extracted.

```csharp
// LinuxIptablesParser.cs:96-98, FirewallLogRegex.cs:280-336
var srcIpMatch = FirewallLogRegex.SourceIpRegex.Match(line);
var dstIpMatch = FirewallLogRegex.DestinationIpRegex.Match(line);
var protocolMatch = FirewallLogRegex.ProtocolRegex.Match(line);
var inInterfaceMatch = FirewallLogRegex.InInterfaceRegex.Match(line);
var outInterfaceMatch = FirewallLogRegex.OutInterfaceRegex.Match(line);
var macMatch = FirewallLogRegex.MacRegex.Match(line);
// ...plus Window, Length, TOS, TTL, ID, Flags, HOPLIMIT, TC, FLOWLBL,
// PREC, RES, URGP, DF, UID, GID, MARK, PHYSIN, PHYSOUT, VPROTO,
// VID, SPI, FRAG, MTU, and SuppressedRegex callback detection
```

Lines matching the kernel rate-limit pattern (`N callbacks suppressed`) are tracked in `Warnings[]` before the SRC/DST/PROTO check. Lines missing SRC/DST or PROTO return `null`. Lines that match SRC/DST/PROTO but have invalid port values or unparseable timestamps throw `FormatException` or `ArgumentException`; absent optional port fields default to 0 and absent metadata fields are omitted from `LinuxSpecific`.

### Fail-Soft Per-Line Try/Catch

```csharp
// LinuxIptablesParser.cs:43-75
for (var i = 0; i < lines.Length; i++)
{
    var line = lines[i];
    try
    {
        var evt = ParseLine(line, referenceDate);
        if (evt != null)
            events.Add(evt);
    }
    catch (Exception ex) when (ex is FormatException || ex is ArgumentException)
    {
        var message = $"Failed to parse line: {FirewallLogRegex.SanitizeLineForError(line)} Error: {ex.Message}";
        errors.Add(message);
        _logSink.Write(LogLevel.Warning, message, ex);
    }
}
```

The exception filter (`when`) restricts catching to parsing errors — unexpected exceptions propagate normally.

### Action Derivation

**iptables:** extracts the text before the first key=value field and checks for ACCEPT/DROP/REJECT tokens:

```csharp
// LinuxIptablesParser.cs:143-168
private static string DeriveAction(string line)
{
    var prefix = FirewallLogRegex.ExtractPrefix(line);
    if (string.IsNullOrWhiteSpace(prefix))
        return UnknownAction;

    var upper = prefix.ToUpperInvariant();
    if (FirewallLogRegex.IsWholeToken(upper, "ACCEPT"))
        return "ACCEPT";
    if (FirewallLogRegex.IsWholeToken(upper, "DROP"))
        return "DROP";
    if (FirewallLogRegex.IsWholeToken(upper, "REJECT"))
        return "REJECT";
    return UnknownAction;
}
```

**nftables:** derives action from the chain name (e.g., `INPUT_DROP` → `DROP`, `FORWARD` → `UNKNOWN`).

### Immutable Validated Event Model

`UnifiedEvent` uses `required` and `init` setters to enforce immutability and mandatory field assignment on core fields (timestamp, IPs, protocol, log format). Optional fields like `SourcePort` and `Action` have sensible defaults:

```csharp
// UnifiedEvent.cs:47-51
public required string SourceIP
{
    get => _sourceIP;
    init => _sourceIP = ValidateIp(value, nameof(SourceIP));
}
```

`ValidateIp` first applies a strict IPv4 regex to reject overflow octets and short forms (e.g., `999.999.999.999`, `1.2.3`), then falls back to `IPAddress.TryParse()` for IPv6 and edge-case formats. Invalid IPs throw at construction, not during detection.

### Extensible Metadata Dictionary

```csharp
// UnifiedEvent.cs:114-126
public IReadOnlyDictionary<string, string> LinuxSpecific
{
    get => _linuxSpecific;
    init
    {
        if (value == null)
            throw new ArgumentNullException(nameof(LinuxSpecific));
        _linuxSpecific = new ReadOnlyDictionary<string, string>(
            new Dictionary<string, string>(value));
    }
}
```

The defensive copy ensures the caller's dictionary cannot be mutated to affect the event after construction.

### Error Sanitization

```csharp
// FirewallLogRegex.cs:241-278
internal static string SanitizeLineForError(string line)
{
    // Truncate to 200 chars, replace control chars, normalize whitespace control chars to spaces
    foreach (var ch in line)
    {
        if (ch == '\r' || ch == '\n' || ch == '\t')
            builder.Append(' ');
        else if (char.IsControl(ch))
            builder.Append('?');
        else
            builder.Append(ch);
    }
}
```

## Security Takeaways

1. Multi-regex extraction prevents one failed field from blocking extraction of all other fields on the same line
2. The fail-soft try/catch with filtered exception types ensures unexpected errors surface rather than being silently swallowed
3. Immutable events with defensive dictionary copies prevent downstream code from corrupting parsed data
4. Error sanitization strips control characters and truncates to 200 chars, preventing log injection through malicious log lines
5. The null logging default (`NullLogSink.Instance`) ensures the parser works correctly even when no logging infrastructure is configured
